# Evaluation Report

Dataset: **dair-ai/emotion**, remapped to six tones (see `data/label_map.py`).
Training data capped at 2,500 samples/class (`--max-per-class 2500`) → 9,432 rows.
Held-out evaluation uses the dataset's **own test split** (2,000 rows) — data the
model never saw during training.

Reproduce:
```bash
python -m src.download_data --max-per-class 2500   # train.csv
python -m src.download_data --split test           # test.csv
python -m src.train --model logreg
python -m src.evaluate
```

## Headline
- **Held-out macro-F1: 0.866** (TF-IDF 1–2 grams + Logistic Regression, `class_weight="balanced"`)
- Held-out accuracy: 0.91 (2,000 rows)
- Cross-validated macro-F1 during training: 0.879 (+/- 0.011)

### Model comparison (5-fold CV macro-F1, from `notebooks/01_eda.ipynb`)
| Model | macro-F1 | Notes |
|---|---|---|
| Linear SVM | **0.902** | highest, but **no `predict_proba`** |
| Logistic Regression | 0.879 | **chosen** — gives confidence scores the app needs |
| Naive Bayes | 0.588 | weak here; kept only as a baseline |

**Decision:** we ship **Logistic Regression**, not the marginally-better SVM. The
product needs per-class probabilities (confidence score + score bar chart), and a
~2-point F1 gap does not justify losing probability outputs.

## Per-class (held-out test)
| Tone | Precision | Recall | F1 | Support |
|---|---|---|---|---|
| anger | 0.85 | 0.88 | 0.87 | 275 |
| anxiety | 0.83 | 0.90 | 0.86 | 202 |
| loneliness | 0.79 | 0.90 | 0.84 | 30 |
| neutral | 0.95 | 0.93 | 0.94 | 907 |
| sadness | 0.93 | 0.89 | 0.91 | 551 |
| stress | 0.69 | 0.89 | 0.78 | 35 |

- **stress** is the weakest (precision 0.69): it is keyword-derived from a source
  with no native "stress" label, so it overlaps with anxiety. Smallest support, too.
- **loneliness** (also keyword-derived, n=30) holds up better but is data-starved.
- neutral/sadness are strongest — most data, most distinct vocabulary.

## Confusion matrix
See `reports/confusion_matrix.png` (generated by `python -m src.evaluate`). Expect
blur between **anxiety↔stress** and **sadness↔loneliness** — those tones genuinely
overlap in language, and the derived classes share vocabulary with their parents.

## Distress safety layer (reported SEPARATELY)
- Approach: rule-based regex (`src/distress.py`), run on **raw** text, tuned for **recall**.
- Probe-set recall: **1.00** (5/5 true-distress caught, 0 false negatives, 0 false alarms).
- Stated trade-off: prioritizes recall; **will** produce false positives in the wild
  and **will** miss indirect/coded language. It is a safety net, not a guarantee, and
  is intentionally **not** an ML model trained on real crisis data.

## Honest limitations
- `joy`/`love`/`surprise` were collapsed into `neutral`, so "neutral" really means
  "not a negative tone" in this app.
- `stress` and `loneliness` are keyword-derived → noisier and data-starved; real
  labeled examples are the top data-collection priority for v2.
- English social-media text only → biased toward that register and demographic.
- TF-IDF matches words, not meaning (no sarcasm/context understanding).
